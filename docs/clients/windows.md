# Windows 客户端指南

## 先判断你是哪条接入路径

### 路径 A：单台 VPS 独立部署

这时继续使用 `./scripts/show_info.sh` 输出的单节点分享链接。

### 路径 B：已发布订阅的受管环境

如果你的环境已经有统一订阅入口、多节点池或运维侧生成的客户端配置，那么应优先使用订阅 URL / 导入链接。

当前这份文档的其余内容，主要描述路径 A 的基线思路；它不是“订阅发布环境”的权威接入文档。

## 推荐客户端

### 路径 A：单台 VPS 独立部署

- `sing-box for Windows`
- 只有在你明确维护一份转换配置时，才建议使用兼容 `Clash Meta` 的客户端

### 路径 B：已发布订阅的受管环境

- `Hiddify` 优先
- `v2rayN` 作为备用

## 推荐路径

路径 A 下，优先使用生成出的 VLESS + Reality 链接。

如果是路径 B，请直接导入订阅，不要再手工维护单节点链接。

## Windows 使用检查清单

1. 路径 A：从 `./scripts/show_info.sh` 输出中导入 VLESS 链接。
2. 路径 B：直接导入运维侧发布的订阅链接。
3. 如果你使用的是单节点分享链接，确认服务器地址指向 VPS 公网 IP。
4. 如果你使用的是单节点分享链接，确认 `sni`、`pbk`、`sid` 与渲染后的配置一致。
5. 按需求决定只给浏览器走代理，还是启用系统级隧道模式。

## 路径 B 下的节点切换

如果你导入的是订阅而不是单节点链接，客户端通常会先选中订阅里的第一个节点。

在当前这套现网订阅里，节点顺序是：

1. `GG-Lisa-Stable`
2. `GG-Dedirock`
3. `GG-Akile`

所以 Hiddify 首次导入后默认落在 `Lisa`，这是当前订阅内容顺序导致的，不是客户端出错。

如果你想切换节点：

1. 打开 Hiddify。
2. 进入当前订阅或节点列表页。
3. 先执行一次延迟测试。
4. 直接点选你要的节点，例如 `GG-Dedirock` 或 `GG-Akile`。

如果你想让默认节点不是 `Lisa`，就不是客户端操作问题了，而是要调整订阅文件里的节点顺序或改成带分组/自动选择的订阅模型。

## 回退路径

- SOCKS5：`BASE_PORT`
- HTTP：`BASE_PORT + 1`

这两个入口只建议用于调试，或者客户端本身不支持 Reality 的场景。

## 验证方式

- 检查浏览器的出口 IP。
- 如果你还要覆盖开发工具，再选一个 CLI 工具通过代理验证一次。
