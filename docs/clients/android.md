# Android 客户端指南

## 先判断你是哪条接入路径

### 路径 A：单台 VPS 独立部署

这时继续使用 `./scripts/show_info.sh` 输出的单节点分享链接。

### 路径 B：已发布订阅的受管环境

如果你的环境已经有统一订阅入口，那么 Android 客户端应优先导入订阅，而不是继续手工维护单条 VLESS 链接。

当前这份文档的其余内容，主要描述路径 A 的基线方法。

## 推荐客户端

- `Hiddify`
- `v2rayNG`
- 只有在你明确维护 sing-box 原生 remote profile 时，再考虑 `sing-box` Android

## 推荐导入路径

路径 A 下，优先使用 VLESS + Reality。

部署完成后，路径 A 在 VPS 上执行 `./scripts/show_info.sh` 获取生成出的分享链接。

如果是路径 B，请直接导入订阅链接。

如果你的运维环境还额外发布了“单节点订阅链接”，那比“先导入多节点再手工点选”更直接。

## Android 使用检查清单

1. 路径 A：导入 VLESS Reality 分享链接。
2. 路径 B：导入订阅链接。
3. 如果你使用的是单节点分享链接，确认服务器地址是 VPS 公网 IP。
4. 如果你使用的是单节点分享链接，确认端口是 `BASE_PORT + 3`。
5. 如果你使用的是单节点分享链接，确认链接中带有 `pbk`、`sid` 和 `sni`。
6. 如需整机代理，再按需要开启 route 模式。

## 验证方式

- 用该配置打开浏览器。
- 确认出口 IP 已变成 VPS IP。
- 确认目标站点访问正常，且没有 TLS 证书警告。

## 安全提示

- 如果怀疑分享链接泄露，优先轮换 UUID。
- Reality 密钥只应在你准备同步更新所有客户端时再轮换。
